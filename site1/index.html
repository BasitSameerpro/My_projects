API Reference

Endpoints

Base URL: https://pakistan-weather-api.vercel.app

GET /api/v1/weather/:city Current weather
GET /api/v1/forecast/:city 5-day forecast
GET /api/v1/cities All supported cities
GET /api/v1/cities/search?q= Search cities
GET /api/v1/health Health check

Query Parameters

PARAM ?format=json | text | csv Output format
PARAM ?days=7 Forecast days (1-16)
KEY ?apikey=YOUR_KEY Use your own WeatherAPI.com key
Output

Three Formats, One Endpoint

Every weather endpoint supports JSON, text, and CSV output. Just add ?format= to switch.

GET /api/v1/weather/karachi
{ "ok": true, "city": "Karachi", "province": "Sindh", "current": { "temperature_c": 25, "condition": "Clear Sky", "condition_icon": "☀️", "wind_speed_kmh": 23, "humidity_percent": 72 }, "source": "Open-Meteo" }

Best for: web apps, mobile apps, API integrations

GET /api/v1/weather/karachi?format=text
Karachi, Sindh, Pakistan | 25°C (Feels 26°C) | ☀️ Clear Sky | Wind: 23 km/h WSW | Humidity: 72% | UV: 2

Best for: AI agents, CLI tools, chatbots, LLM function calling

For AI Agents

CLI Tool

The CLI calls Open-Meteo directly — no server needed. AI agents can run it as a terminal command and parse the one-line response.

Terminal
# Current weather (one-liner — perfect for AI parsing) $ pk-weather lahore Lahore, Punjab, Pakistan | 26°C (Feels 27°C) | 🌤️ Mainly Clear | Wind: 3 km/h ESE | Humidity: 42% # JSON output $ pk-weather karachi --json # 5-day forecast $ pk-weather islamabad --forecast # Custom forecast range $ pk-weather quetta --forecast --days 7 # Search cities $ pk-weather --search mar 🔍 Mardan, Lakki Marwat # List all 150+ cities $ pk-weather --list

Install Globally

$ npm install -g pakistan-weather-api
Optional

Bring Your Own API Key

Want to use WeatherAPI.com instead? Pass your free API key and it becomes the primary source.

HTTP Request
# Via query parameter GET /api/v1/weather/lahore?apikey=YOUR_KEY # Via header curl -H "X-API-Key: YOUR_KEY" https://pakistan-weather-api.vercel.app/api/v1/weather/lahore
